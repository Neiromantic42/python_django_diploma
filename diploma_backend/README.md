# 🛒 Интернет-магазин на Django

Дипломный проект — полнофункциональный интернет-магазин с каталогом товаров, корзиной, оформлением заказов, асинхронной оплатой и личным кабинетом.

Функционал аналогичен популярным маркетплейсам: **holodilnik.ru**, **citilink.ru**, **ozon.ru**.

## 🛠 Технологии

- **Python 3.11+**
- **Django 5.0+** — основной фреймворк
- **Django REST Framework** — REST API
- **Celery 5 + Redis** — асинхронная обработка оплаты
- **PostgreSQL / SQLite** — база данных
- **django-phonenumber-field** — валидация номеров телефонов
- **Pillow** — обработка изображений
- **Black, isort, autoflake** — форматирование кода (PEP8)
- **Docker** — контейнеризация (опционально)

## ✨ Функционал

### 🏠 **Главная страница:**
- Топ товары по популярности
- Избранные категории
- Товары ограниченного тиража (баннеры)

### 📦 **Каталог:**
- Фильтрация (цена, наличие, доставка, теги)
- Сортировка (цена, популярность, отзывы, новизна)
- Постраничная навигация
- Поиск товаров

### 🛍️ **Товар:**
- Детальная информация
- Галерея изображений
- Отзывы покупателей (с рейтингом 1-5)
- Технические характеристики
- Добавление в корзину

### 🛒 **Корзина:**
- Добавление/удаление товаров
- Изменение количества
- Автоматический пересчет стоимости

### 📝 **Оформление заказа:**
1. Корзина → проверка товаров
2. Данные покупателя (ФИО, телефон, email, адрес)
3. Выбор доставки (обычная 200₽, экспресс 500₽, бесплатная от 2000₽)
4. Выбор способа оплаты (онлайн карта)
5. **Асинхронная оплата через Celery** (имитация платежного шлюза)

### 💳 **Оплата:**
- Валидация данных карты (номер, срок, CVV, ФИО держателя)
- Бизнес-логика проверки (четность номера, последняя цифра)
- Статусы: `pending` → `success` / `failed`
- История попыток оплаты

### 👤 **Личный кабинет:**
- Профиль (редактирование ФИО, телефона, аватара, пароля)
- История заказов со статусами
- Детали заказа (товары, стоимость, доставка, оплата)

### 🔐 **Авторизация:**
- Регистрация (`POST /api/sign-up`)
- Вход (`POST /api/sign-in`)
- Выход (`POST /api/sign-out`)
- Сессионная авторизация (Django Session)

### ⚙️ **Админ-панель (Django Admin):**
- Управление пользователями и профилями
- Товары (категории, теги, характеристики, изображения)
- Заказы и оплаты
- Отзывы и рейтинги
- Настройки доставки

## 🚀 Установка и запуск

### 1️⃣ **Клонирование репозитория:**
```bash
git clone https://github.com/Neiromantic42/python_django_diploma.git
cd python_django_diploma/diploma_backend
```

### 2️⃣ **Создание виртуального окружения:**
```bash
python -m venv .venv
source .venv/bin/activate  # Linux/Mac
# .venv\Scripts\activate   # Windows
```

### 3️⃣ **Установка зависимостей:**
```bash
pip install -r requirements.txt
```

### 4️⃣ **Настройка переменных окружения (.env):**

Создайте файл `.env` в корне `diploma_backend/`:
```env
DEBUG=True
SECRET_KEY='your-secret-key-here'
ALLOWED_HOSTS=127.0.0.1,localhost

# База данных (SQLite для разработки)
DATABASE_URL=sqlite:///db.sqlite3

# Redis для Celery
REDIS_URL=redis://localhost:6379/0
CELERY_BROKER_URL=redis://localhost:6379/0
CELERY_RESULT_BACKEND=redis://localhost:6379/0
```

### 5️⃣ **Применение миграций:**
```bash
python manage.py migrate
```

### 6️⃣ **Загрузка тестовых данных (фикстуры):**
```bash
python manage.py loaddata fixtures/full_db.json
```

**Фикстуры включают:**
- Администратора: `admin / 123`
- Тестовых пользователей (см. таблицу ниже)
- Категории товаров
- 10+ товаров с изображениями
- Отзывы и рейтинги
- Настройки доставки

### 7️⃣ **Создание суперпользователя (опционально):**
```bash
python manage.py createsuperuser
```

### 8️⃣ **Запуск сервера разработки:**
```bash
python manage.py runserver 0.0.0.0:8000
```

Сайт доступен: **http://127.0.0.1:8000**  
Админка: **http://127.0.0.1:8000/admin**

## ⚡ Celery (асинхронные задачи)

Для обработки оплаты используется **Celery** с **Redis** в качестве брокера сообщений.

### **Запуск Redis:**
```bash
# Linux/Mac
sudo systemctl start redis

# Docker
docker run -d -p 6379:6379 redis
```

### **Запуск Celery Worker:**
```bash
celery -A diploma_backend worker -l info
```

### **Задачи:**

- `payment.tasks.process_payment` — обработка оплаты с имитацией задержки (3 секунды)

**Флоу оплаты:**
1. Пользователь отправляет данные карты → `POST /api/payment/{id}`
2. Django создает `Payment` со статусом `pending`
3. Задача добавляется в очередь Celery
4. Ответ пользователю: *"Ждём подтверждения оплаты"*
5. Celery Worker обрабатывает оплату (проверка номера карты)
6. Обновление статуса: `success` / `failed`
7. Фронт проверяет статус через polling или WebSocket

## 📖 API документация

API построен на **Django REST Framework**.

### **Основные эндпоинты:**

#### 🔐 Авторизация:
- `POST /api/sign-up` — регистрация
- `POST /api/sign-in` — вход
- `POST /api/sign-out` — выход

#### 👤 Профиль:
- `GET /api/profile` — получить профиль
- `POST /api/profile` — обновить профиль
- `POST /api/profile/avatar` — загрузить аватар
- `POST /api/profile/password` — сменить пароль

#### 📦 Каталог:
- `GET /api/catalog/` — список товаров (фильтры, сортировка, пагинация)
- `GET /api/products/{id}` — детали товара
- `GET /api/products/popular/` — популярные товары
- `GET /api/products/limited/` — товары ограниченного тиража
- `GET /api/banners/` — баннеры для главной страницы
- `GET /api/categories/` — категории
- `GET /api/tags/` — теги товаров
- `POST /api/products/{id}/reviews` — добавить отзыв

#### 🛒 Корзина:
- `GET /api/basket` — получить корзину
- `POST /api/basket` — добавить товар
- `DELETE /api/basket/{id}` — удалить товар

#### 📝 Заказы:
- `POST /api/orders` — создать заказ из корзины
- `GET /api/orders` — история заказов
- `GET /api/order/{id}` — детали заказа
- `POST /api/order/{id}` — подтвердить заказ (заполнить данные)

#### 💳 Оплата:
- `POST /api/payment/{id}` — оплатить заказ (асинхронно через Celery)

**Swagger документация:** доступна через фронтенд-пакет.

## 🐳 Docker

### **Запуск через Docker Compose:**
```bash
docker-compose up --build
```

**Сервисы:**
- `web` — Django приложение (порт 8000)
- `redis` — брокер для Celery (порт 6379)
- `celery` — воркер для фоновых задач

### **Применение миграций и фикстур в контейнере:**
```bash
docker-compose exec web python manage.py migrate
docker-compose exec web python manage.py loaddata fixtures/full_db.json
```
